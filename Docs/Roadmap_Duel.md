# Roadmap — «Дуэль»

## Этап 0. Подготовка (1–3 дня)

- [х] Создать Unity‑проект (URP, PC).
- [ ] Настроить репозиторий (Git).
- [ ] Создать базовую структуру папок.
- [ ] Добавить `GDD_Duel.md` и `TDD_Duel.md` в `Assets/Docs` или корень репо.

## Этап 1. Базовый прототип дуэли (1–2 недели)

Цель: в серой коробочной сцене можно сыграть один раунд и победить/проиграть.

- [ ] Реализовать `PlayerControllerFP` (движение + камера).
- [ ] Добавить простой `Weapon` (револьвер) с raycast‑стрельбой.
- [ ] Сделать `Health` и `IDamageable`.
- [ ] Реализовать простейшего противника‑манекен (стоит, имеет здоровье).
- [ ] Реализовать `DuelManager` с состояниями:
  - [ ] `Waiting`
  - [ ] `Countdown` (визуальный таймер)
  - [ ] `Fight` (разрешены выстрелы)
  - [ ] `Result` (определение победителя)
- [ ] Связать смерть игрока/бота с `DuelManager` (определение победы/поражения).
- [ ] Добавить базовый HUD с перекрестием и текстами состояния.

## Этап 2. ИИ дуэлянта (1–2 недели)

Цель: против тебя играет бот, реагирует на сигнал с задержкой.

- [ ] Создать `AI_Duelist`:
  - [ ] Реакция на сигнал начала боя от `DuelManager`.
  - [ ] Случайная задержка выстрела в диапазоне.
  - [ ] Стрельба по игроку с допуском по точности.
- [ ] Добавить уровни сложности (параметры задержки и точности).
- [ ] Добавить отображение сложности в UI.
- [ ] Добавить лог/отладку времени реакции бота и игрока.

## Этап 3. Минимальный визуал и звук (2–3 недели)

Цель: прототип становится атмосферным, пригодным для демо.

- [ ] Найти/добавить временные модели рук и револьвера.
- [ ] Настроить анимации рук (idle, выстрел).
- [ ] Добавить выстрельные эффекты (muzzle flash, звук).
- [ ] Добавить базовые звуки окружения.
- [ ] Включить лёгкий пост‑процесс (Bloom, Vignette, Color Grading).
- [ ] Настроить `CameraSway` и отдачу.

## Этап 4. Локации (3–4 недели)

Цель: минимум 2 играбельные локации.

- [ ] Создать сцены:
  - [ ] `Duel_West` — Dикий Запад.
  - [ ] `Duel_RussiaWinter` — Россия зима.
- [ ] Собрать окружение из ассетов или простых моделей.
- [ ] Настроить освещение и небо для каждой сцены.
- [ ] Добавить VFX (пыль/ветер, снегопад).
- [ ] Синхронизировать стартовые позиции дуэлянтов через `DuelManager`.

## Этап 5. Полировка геймплея (2–3 недели)

Цель: приятное чувство стрельбы и дуэли.

- [ ] Тонкая настройка чувствительности мыши, FOV.
- [ ] Улучшение отдачи и sway оружия.
- [ ] Тонкая настройка времён реакции и точности ИИ.
- [ ] Добавление статистики на экран результата:
  - [ ] Время реакции игрока.
  - [ ] Попадание (голова/тело).
- [ ] Улучшение UI (минималистичный, читабельный).

## Этап 6. Дополнительное оружие и контент (2–4 недели)

Цель: базовое разнообразие.

- [ ] Добавить ещё 1–2 вида оружия (например, дворянский пистолет).
- [ ] Ввести систему `WeaponConfig` (ScriptableObjects).
- [ ] Сделать выбор оружия в меню.
- [ ] Добавить простую прогрессию (разблокировка локаций/оружия).
